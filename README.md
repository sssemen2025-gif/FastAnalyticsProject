# FastAnalyticsProject

**Высокопроизводительные аналитические алгоритмы на C++, доступные из Python через pybind11**

---

## О проекте

Проект создан в учебных целях и предназначен для портфолио.
Он показывает, как ускорить типовые операции анализа данных, вынеся вычислительно тяжёлые части в скомпилированный C++ код.

Модуль полезен Python-разработчикам и аналитикам данных, которым важно понимать узкие места производительности и уметь применять низкоуровневые оптимизации.

---

## Возможности

- Скользящее среднее и экспоненциальное сглаживание
- Расчёт медианы, Z-оценок и поиск аномалий
- Линейная регрессия с выдачей коэффициентов и R²
- Метрики регрессии (MAE, MSE, RMSE, R²) за один проход
- Поиск локальных пиков
- Существенное ускорение по сравнению с чистым Python и некоторыми библиотеками

---

## Доступные функции

| Функция | Описание |
|---------|----------|
| `moving_average(data, window)` | Скользящее среднее, O(n) |
| `ewma(data, alpha)` | Экспоненциальное сглаживание |
| `median(data)` | Медиана |
| `zscore(data)` | Z-оценки (стандартизация) |
| `find_anomalies(data, threshold)` | Поиск выбросов по Z-оценкам |
| `linear_regression(x, y)` | Линейная регрессия (intercept, slope, R²) |
| `regression_metrics(y_true, y_pred)` | MAE, MSE, RMSE, R² за один проход |
| `find_peaks(data, lookahead, min_height)` | Поиск локальных максимумов |

C++ модуль компилируется в `.pyd` (динамическая библиотека), которую Python загружает как обычную библиотеку.
При вызове функций pybind11 автоматически конвертирует Python-списки в `std::vector<double>`, выполняет нативный код и возвращает результат обратно в Python.

## Анализ производительности

**Сильные стороны:**

- `find_peaks`, `find_anomalies`, `linear_regression`, `regression_metrics` -- ускорение от 15 до 46 раз
- Все перечисленные функции делают несколько операций за один проход по данным
- Минимальные накладные расходы на выделение памяти

**Слабые стороны:**

- `moving_average` -- NumPy быстрее за счёт векторизации (SIMD-инструкции)
- `median` -- NumPy использует частичную сортировку (O(n)), мой код делает полную сортировку (O(n log n))

**Вывод:**

Модуль хорошо подходит для задач, где нужна высокая скорость без тяжёлых зависимостей.
В перспективе -- добавление прямой поддержки numpy-массивов, что уберёт затраты на конвертацию списков и может приблизить к NumPy по скользящему среднему и медиане.
